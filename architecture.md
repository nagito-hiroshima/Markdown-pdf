# Markdown PDF Viewer 拡張機能 要件定義書

## 1. 概要

本システムは、GitHub Raw URLなどで公開されているMarkdown形式のファイルを取得し、ブラウザ上でPDFのように整形表示できる拡張機能である。

Markdownファイルをそのまま読むのではなく、見出し、表、コードブロック、画像、リンクなどを見やすく整形し、必要に応じてPDFとして保存・印刷できることを目的とする。

対象例として、以下のようなURLを想定する。

```text
https://raw.githubusercontent.com/.../docs/design/xxx.md
```

---

## 2. 目的

Markdown形式の設計書や技術資料を、ブラウザ上で読みやすいPDF風レイアウトとして表示する。

主な目的は以下の通りである。

* GitHub RawのMarkdownファイルを見やすく表示する
* MarkdownをPDF風のページレイアウトに変換する
* 印刷やPDF保存をしやすくする
* 技術資料、設計書、仕様書を共有しやすくする
* Vivado、FPGA、ソフトウェア設計などの研究資料を整理して閲覧しやすくする

---

## 3. 対象ユーザー

本拡張機能の対象ユーザーは以下の通りである。

* GitHub上でMarkdownドキュメントを管理している開発者
* 研究資料や設計書をMarkdownで作成している学生・研究者
* MarkdownをPDFとして確認・共有したいユーザー
* ブラウザ上でMarkdown資料をきれいに閲覧したいユーザー

---

## 4. 想定利用シーン

### 4.1 GitHub Raw Markdownの閲覧

ユーザーがGitHub RawのMarkdown URLを開いたとき、拡張機能がMarkdownファイルを検出し、PDF風のプレビュー画面を表示する。

### 4.2 MarkdownファイルのPDF保存

ユーザーが表示中のMarkdown資料をPDFとして保存したい場合、拡張機能のボタンからPDF保存またはブラウザ印刷機能を利用できる。

### 4.3 技術設計書の共有

研究や開発で作成したMarkdown設計書を、見やすいレイアウトで確認し、PDFとして提出・共有できる。

---

## 5. システム構成

本システムはブラウザ拡張機能として実装する。

主な構成は以下の通りである。

| 構成要素              | 内容                         |
| ----------------- | -------------------------- |
| Browser Extension | Chrome / Edge向け拡張機能        |
| Content Script    | 開いているページがMarkdownかどうかを判定する |
| Background Script | URL取得、状態管理、設定管理を行う         |
| Markdown Parser   | MarkdownをHTMLに変換する         |
| PDF Preview UI    | PDF風に整形表示する画面              |
| PDF Export        | 印刷またはPDF保存を行う              |
| Settings          | 表示テーマ、フォント、余白などの設定         |

---

## 6. 機能要件

## 6.1 Markdown検出機能

### 概要

現在開いているページがMarkdownファイルであるかを判定する。

### 要件

* URLの拡張子が `.md` の場合、Markdownファイルとして認識する
* `raw.githubusercontent.com` のURLを対象にする
* Content-Typeが `text/plain` または `text/markdown` の場合もMarkdownとして扱う
* ユーザーが手動で「Markdownとして開く」こともできる

### 対象URL例

```text
https://raw.githubusercontent.com/user/repository/main/docs/design/sample.md
https://example.com/document.md
```

---

## 6.2 Markdown取得機能

### 概要

指定されたURLからMarkdown本文を取得する。

### 要件

* 現在開いているMarkdownページの内容を取得できる
* URLを直接入力してMarkdownを取得できる
* GitHub Raw URLからMarkdownを取得できる
* 取得失敗時はエラーメッセージを表示する
* ネットワークエラー、404、認証エラーに対応する

### エラー例

| エラー              | 表示内容                  |
| ---------------- | --------------------- |
| 404 Not Found    | Markdownファイルが見つかりません  |
| 401 Unauthorized | 認証が必要なファイルです          |
| Network Error    | ファイルの取得に失敗しました        |
| Invalid Markdown | Markdownとして読み込めませんでした |

---

## 6.3 Markdown解析機能

### 概要

取得したMarkdownをHTMLに変換する。

### 対応するMarkdown記法

* 見出し
* 段落
* 箇条書き
* 番号付きリスト
* 太字
* 斜体
* 打ち消し線
* 引用
* コードブロック
* インラインコード
* 表
* リンク
* 画像
* 水平線
* チェックボックス
* GitHub Flavored Markdown

### コードブロック

コードブロックは言語指定に応じてシンタックスハイライトを行う。

例：

```verilog
module top(
    input wire clk,
    output wire led
);
endmodule
```

---

## 6.4 PDF風プレビュー機能

### 概要

MarkdownをPDFのようなレイアウトでブラウザ上に表示する。

### 要件

* A4サイズ相当のページ幅で表示する
* 白背景、黒文字を基本とする
* 見出し、表、コードブロックを読みやすく整形する
* ページ余白を設定できる
* 目次を表示できる
* スクロールで全体を閲覧できる
* 印刷時にレイアウトが崩れないようにする

### 表示項目

* タイトル
* 本文
* 見出し
* 表
* コードブロック
* 画像
* ページ番号
* 更新日時または取得日時

---

## 6.5 目次生成機能

### 概要

Markdown内の見出しをもとに目次を自動生成する。

### 要件

* `#`、`##`、`###` などの見出しを検出する
* サイドバーに目次を表示する
* 目次をクリックすると該当箇所へ移動する
* PDF保存時に目次を含めるか選択できる

---

## 6.6 PDF出力機能

### 概要

表示中のMarkdownプレビューをPDFとして保存できるようにする。

### 要件

* ブラウザの印刷機能を利用してPDF保存できる
* 「PDFとして保存」ボタンを用意する
* 印刷用CSSを適用する
* A4縦向きに対応する
* ページ分割時に見出しや表が崩れにくいようにする
* ファイル名はMarkdownファイル名をもとに自動設定する

### ファイル名例

```text
hdmi_ddr3_pipeline_design.pdf
```

---

## 6.7 表示設定機能

### 概要

ユーザーが表示スタイルを変更できる。

### 設定項目

| 項目      | 内容           |
| ------- | ------------ |
| テーマ     | Light / Dark |
| 用紙サイズ   | A4 / Letter  |
| フォントサイズ | 小 / 標準 / 大   |
| 余白      | 狭い / 標準 / 広い |
| コード折り返し | ON / OFF     |
| 目次表示    | ON / OFF     |
| ページ番号   | ON / OFF     |

---

## 6.8 GitHub連携補助機能

### 概要

GitHub上のMarkdownファイルをより扱いやすくする。

### 要件

* `raw.githubusercontent.com` のURLを自動判定する
* GitHubの通常ページURLからRaw URLへ変換できる
* GitHub上の `.md` ファイルを開いたときに「PDF表示で開く」ボタンを表示する
* リポジトリ名、ブランチ名、ファイルパスを取得してタイトル表示に利用する

### 変換例

通常URL：

```text
https://github.com/user/repo/blob/main/docs/sample.md
```

Raw URL：

```text
https://raw.githubusercontent.com/user/repo/main/docs/sample.md
```

---

## 6.9 ローカルMarkdown表示機能

### 概要

ユーザーのPC上にあるMarkdownファイルを読み込み、PDF風に表示できる。

### 要件

* `.md` ファイルをアップロードできる
* ドラッグ&ドロップで読み込める
* 読み込んだ内容をブラウザ上でプレビューできる
* ローカルファイルの内容は外部に送信しない

---

## 7. 非機能要件

## 7.1 対応ブラウザ

初期対応は以下とする。

* Google Chrome
* Microsoft Edge

将来的に以下も検討する。

* Firefox
* Brave

---

## 7.2 性能要件

* 1MB程度のMarkdownファイルを3秒以内に表示できること
* 画像を含むMarkdownでも大きく表示崩れしないこと
* 100ページ相当のMarkdownでもスクロール操作が極端に重くならないこと

---

## 7.3 セキュリティ要件

* 外部MarkdownのHTMLをそのまま実行しない
* `<script>` タグなど危険なHTMLは無効化する
* ローカルファイルの内容を外部サーバへ送信しない
* 認証トークン付きURLをログに保存しない
* 保存履歴に機密URLを残さない設定を用意する
* GitHubのプライベートリポジトリURLを扱う場合は注意喚起を表示する

---

## 7.4 プライバシー要件

* ユーザーが閲覧したMarkdown URLを外部に送信しない
* 利用履歴の保存は任意とする
* ローカル変換を基本とする
* PDF生成処理はブラウザ内で完結させる

---

## 7.5 保守性要件

* Markdown変換処理、UI、PDF出力処理を分離して実装する
* CSSテーマを変更しやすい構成にする
* 将来的にVS Code拡張やWebアプリ版へ展開できる構成にする

## 10. 権限要件

Chrome拡張機能として、以下の権限を想定する。

```json
{
  "permissions": [
    "activeTab",
    "scripting",
    "storage"
  ],
  "host_permissions": [
    "https://raw.githubusercontent.com/*",
    "https://github.com/*"
  ]
}
```

### 権限の目的

| 権限               | 用途                     |
| ---------------- | ---------------------- |
| activeTab        | 現在開いているタブのURL取得        |
| scripting        | ページ内容の取得・操作            |
| storage          | 表示設定の保存                |
| host_permissions | GitHub Raw Markdownの取得 |

## 12. 処理フロー

## 12.1 GitHub Raw URLを開いた場合

```text
1. ユーザーがMarkdown URLを開く
2. 拡張機能がURLを判定する
3. Markdown本文を取得する
4. MarkdownをHTMLへ変換する
5. PDF風CSSを適用する
6. プレビュー画面を表示する
7. ユーザーが必要に応じてPDF保存する
```

---

## 12.2 ローカルファイルを開いた場合

```text
1. ユーザーがMarkdownファイルを選択する
2. ファイル内容を読み込む
3. MarkdownをHTMLへ変換する
4. PDF風CSSを適用する
5. プレビュー画面を表示する
6. ユーザーが必要に応じてPDF保存する
```

## 13. エラー処理

| 状況               | 対応                |
| ---------------- | ----------------- |
| Markdownを取得できない  | エラーメッセージを表示       |
| URLがMarkdownではない | 手動変換ボタンを表示        |
| Markdownの解析に失敗   | 元のMarkdownを表示する   |
| 画像が読み込めない        | 代替テキストを表示         |
| PDF保存に失敗         | 印刷機能の利用を案内        |
| 認証が必要なURL        | ログインまたはアクセス権確認を案内 |

---

## 14. 制約事項

* ブラウザ拡張機能のため、対応ブラウザに依存する
* GitHubのプライベートリポジトリはアクセス権が必要である
* トークン付きURLは漏洩リスクがあるため、履歴保存しない
* 複雑なHTMLを含むMarkdownは完全には再現できない可能性がある
* Mermaidなどの図表記法は初期版では任意対応とする

## 19. 成功条件

本拡張機能の成功条件は以下とする。
* GitHub RawのMarkdownファイルをブラウザ上で見やすく表示できる
* 見出し、表、コードブロックが正しく整形される
* PDF保存または印刷が問題なく行える

* トークン付きURLなどの機密情報を不用意に保存しない

## 20. まとめ

本拡張機能は、Markdownで作成された技術資料や設計書を、ブラウザ上でPDF風に表示するためのツールである。

特に、GitHubで管理している設計ドキュメントをそのままきれいに閲覧・PDF保存できるため、研究資料、開発ドキュメント、授業資料、仕様書の確認や共有に有効である。

初期版ではGitHub Raw Markdownの表示とPDF保存を中心に実装し、将来的にはMermaid、数式、表紙、複数ファイル結合などに対応する。
